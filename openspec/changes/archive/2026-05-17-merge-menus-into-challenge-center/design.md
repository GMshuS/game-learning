## Context

当前主菜单（MainMenu.vue）包含5个选项：冒险模式、经营商店、挑战中心、成就系统、设置。挑战中心（GameHall.vue）已有3个子模式：速算竞技场、数学工坊、卡牌对战。冒险模式和经营商店与挑战中心内的玩法属于同一类别（挑战/玩法类），但在主菜单中作为独立入口存在，导致菜单冗长且分类不清晰。

技术栈：Vue 3 Composition API + Vite 5，使用事件驱动视图切换（GameApp.vue 的 currentView 状态管理）。

## Goals / Non-Goals

**Goals:**
- 简化主菜单为3个选项：挑战中心、成就系统、设置
- 将冒险模式和经营商店作为子入口整合到挑战中心
- 保持冒险模式和经营商店原有功能不变，仅改变入口位置
- 保持现有数据流和状态管理不变

**Non-Goals:**
- 不修改冒险模式（AdventureMap/BattleGame）的内部逻辑
- 不修改经营商店（ShopView/CashierGame）的内部逻辑
- 不修改成就系统和设置功能
- 不引入新的状态管理或路由方案

## Decisions

### Decision 1: 事件驱动导航保持不变

**方案**: 继续使用 GameApp.vue 的 currentView + emit 事件模式，不引入 Vue Router。

**理由**: 项目当前使用简单的事件驱动视图切换，无路由依赖。保持现有模式最小化变更范围，避免引入不必要的复杂性。

**替代方案**: 引入 Vue Router 实现真正的路由导航。但这对本次菜单重组来说过度设计，且会增加迁移风险。

### Decision 2: 冒险模式和经营商店作为独立视图保留

**方案**: AdventureMap.vue 和 ShopView.vue 作为独立组件保留，从 GameHall 点击后通过 emit 事件切换到对应视图。

**理由**: 这两个模式已有完整的子流程（冒险→战斗、商店→收银），作为独立视图更符合当前的组件架构。

**替代方案**: 将冒险模式和商店模式嵌入 GameHall 内部作为子组件。但这会破坏现有的视图切换模式，且增加 GameHall 的复杂度。

### Decision 3: 挑战中心卡片布局扩展为5列/自适应

**方案**: GameHall 的卡片布局从3个扩展到5个，使用现有的 `grid-template-columns: repeat(auto-fit, minmax(250px, 1fr))` 自适应布局。

**理由**: 现有 CSS Grid 布局已支持自适应，无需修改样式逻辑即可容纳更多卡片。

## Risks / Trade-offs

- **[风险]** 主菜单选项减少可能降低冒险模式和经营商店的可见度 → **缓解**: 在挑战中心中保持醒目的卡片设计，与现有卡片风格一致
- **[风险]** GameHall 卡片增多在小屏幕上可能需要滚动 → **缓解**: 现有布局已支持 overflow-y: auto，移动端为单列布局
- **[权衡]** 从主菜单进入冒险/商店需要多一次点击 → 可接受，因为挑战中心作为统一入口提供了更好的功能分类
