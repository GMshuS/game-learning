## Context

冒险模式和经营商店已通过 `GameTutorial.vue` 组件实现了玩法说明功能，使用 `❓ 玩法说明` 按钮触发弹窗。速算竞技场、数学工坊、卡牌对战三个挑战中心模式目前缺少该机制。三个组件均为 Vue 3 Composition API 结构，GameTutorial 组件接受 `title` 和 `steps` 属性，通过 `@close` 事件关闭。

## Goals / Non-Goals

**Goals:**
- 三个挑战模式各增加一个 `❓ 玩法说明` 按钮，点击弹出 GameTutorial 教程弹窗
- 教程内容覆盖各模式的核心玩法、规则和操作方式
- 保持与现有冒险模式、经营商店一致的 UI 风格和交互模式

**Non-Goals:**
- 不修改 GameTutorial 组件本身（已满足需求）
- 不实现首次进入自动弹出教程（当前冒险/商店模式也未实现）
- 不添加教程进度追踪或"不再显示"功能

## Decisions

### 1. 复用 GameTutorial 组件

三个模式均直接 import 并使用现有的 `GameTutorial.vue`，传入模式特定的 `title` 和 `steps` 数组。无需创建新的教程组件。

**替代方案**: 为每个模式创建独立教程组件 — 拒绝，因为 GameTutorial 已足够灵活且被两个模式验证可用。

### 2. 教程按钮位置和样式

遵循现有模式：在组件模板顶部添加按钮，使用与 BattleGame.vue 和 ShopView.vue 相同的 `btn-help` 类名和 `❓ 玩法说明` 文案。按钮通过 `@click="showTutorial = true"` 触发。

### 3. 教程步骤设计

每个模式的教程设计为 5-6 个步骤，覆盖：
- **速算竞技场**: 模式选择、三种子模式规则（基础速算/闪电抢答/生存模式）、计分和评级
- **数学工坊**: 答题获取材料、制作物品、上架售卖、销售结算
- **卡牌对战**: 卡组构建、卡牌类型、对战流程、方程卡牌、难度选择

## Risks / Trade-offs

- [教程内容可能过长] → 控制在 5-6 步内，每步描述简洁，GameTutorial 支持滚动
- [移动端按钮可能遮挡] → 按钮使用绝对定位或内联布局，与现有冒险模式保持一致
